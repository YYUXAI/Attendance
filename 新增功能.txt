新增功能开发任务清单（可执行）

说明：
1. 本清单按“先后顺序”执行，默认每项都要有代码、测试与验收。
2. 术语统一：工号、签到、签退、离岗、返岗、月休。
3. 时间统一使用 24 小时制，存储 UTC，展示按班次时区。

=====================================
里程碑 M1：数据层与规则底座
=====================================

[ ] T1. 新增班次配置表 `employee_shift_config`
- 字段：english_name, employee_id(唯一), shift_time_range, shift_checkin_time, shift_checkout_time, monthly_rest_days
- 输出：DDL + repository + 基础 CRUD
- 验收：可新增/查询/更新/删除，employee_id 唯一约束生效

[ ] T2. 新增离返岗记录表（建议 `temporary_leave_records`）
- 字段：employee_id, leave_at, back_at, duration_minutes, reason, remark_required, chat_id, created_by_tg_id
- 输出：DDL + repository
- 验收：能记录“离岗未返岗”状态，返岗后闭合并计算时长

[ ] T3. 实现月休解析与判定工具
- 规则：`2,3,4,5` 对应当月日期；超出当月天数自动忽略
- 输出：service 工具函数（is_monthly_rest_day）
- 验收：单元测试覆盖 28/29/30/31 天月份

=====================================
里程碑 M2：Bot 交互与权限
=====================================

[ ] T4. 更新底部按钮
- 按钮：签到、签退、离岗、返岗、导出、班次
- 验收：普通用户只能看到前四个；权限用户可见全部按钮

[ ] T5. 权限控制（展示 + 后端双校验）
- 来源：admin_list（或独立权限表）
- 规则：无权限触发「班次/导出」时，返回“无权限操作”
- 验收：绕过前端回调也无法执行

[ ] T6. 签到/签退快捷模板
- 点击后自动填充：
  #打卡
  英文名：
  工号：
  事项：签到/签退
- 验收：模板文本正确，字段名统一“工号”

[ ] T7. 离岗按钮逻辑
- 自动填充离岗模板
- 自动识别操作者绑定工号
- 时间自动取系统当前时间
- 验收：离岗记录成功写库，时间不可手改

[ ] T8. 返岗按钮逻辑
- 自动填充返岗模板
- 自动匹配最近未闭合离岗记录
- 自动计算离岗时长
- 验收：返岗后记录闭合，时长正确

[ ] T9. 离岗说明规则
- 时长 > 30 分钟：说明/原因必填
- 时长 <= 30 分钟：说明可空
- 验收：规则拦截有效，错误提示清晰

=====================================
里程碑 M3：班次管理与导出
=====================================

[ ] T10. 班次管理页面/流程
- 点击「班次」弹出配置表（可修改）
- 顶部按钮：取消、上传
- 上传前二次确认（取消/确认）
- 验收：确认后更新数据库；取消不落库

[ ] T11. 导出按钮（私聊 + 权限用户）
- 仅私聊展示；群聊不展示
- 点击发送当天明细文件（CSV/Excel）
- 验收：非私聊或无权限无法导出

[ ] T12. 导出字段与状态展示
- 必含：签到/签退、离岗/返岗、离岗时长、迟到/早退、缺卡、月休
- 月休日展示：月休（免打卡）
- 验收：字段齐全，状态口径一致

=====================================
里程碑 M4：每晚统计（按群隔离）
=====================================

[ ] T13. 定时任务（默认 23:00）
- 每天固定时间触发统计
- 时区可配置（默认 Asia/Shanghai）
- 验收：任务按时触发，重启后不中断

[ ] T14. 按打卡群分发统计
- 依据 `shifts.attendance_group_id`
- 每个群只发本群统计，不跨群
- 验收：A 群不看到 B 群数据

[ ] T15. 每晚统计内容
- 分组：缺卡、迟到、早退、全部正常、月休
- 空分组显示“无”
- 全部无异常时显示“今日无异常，全部正常/休息”
- 验收：消息文案完整且可读

[ ] T16. 群统计与导出范围区分
- 群统计：仅本群
- 班次/导出：权限用户可看全局（所有群）
- 验收：范围边界符合产品要求

=====================================
里程碑 M5：考勤判定口径
=====================================

[ ] T17. 迟到规则
- 实际上班时间 > 班次上班时间 => 迟到
- 验收：边界值（等于/晚 1 分钟）正确

[ ] T18. 早退规则
- 实际下班时间 < 班次下班时间 => 早退
- 验收：边界值（等于/早 1 分钟）正确

[ ] T19. 缺卡规则
- 当天无有效上/下班打卡且非月休 => 缺卡
- 验收：月休日不误判缺卡

[ ] T20. 月休规则
- 月休日免打卡，不计异常
- 月休日有打卡：可保留记录并标记“休息日打卡”（最终展示可配置）
- 验收：统计与导出口径一致

=====================================

=====================================

[ ] T21. 单元测试
- 月休解析、迟到早退、缺卡、离返岗时长、30 分钟说明规则
- 验收：关键函数覆盖率达标

[ ] T22. 集成测试
- 群内按钮流程、私聊导出、23:00 定时群发、权限拦截
- 验收：主流程全通过

[ ] T23. 回归测试
- 现有注册/打卡/OCR 不回退
- 验收：历史功能可用

[ ] T24. 配置与文档
- 增加 .env 配置项（定时、时区、导出格式、权限）
- 更新产品文档与运维手册
- 验收：新同事按文档可独立部署
