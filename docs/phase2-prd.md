# 考勤 Bot 二期 PRD

- 文档版本：2026-06-17
- 产品：@zpxinbot
- 状态：评审稿

## 1. 项目背景

考勤 Bot 一期已上线并在 UX 组生产使用，已具备：

- 群内 TIME.IS 截图打卡（线上大模型识图）
- Telegram 注册绑定（英文名 + 工号）
- Google 排班表同步到 `employee_shift_config`
- 迟到 / 早退 / 缺卡 / 月休判定
- 每晚统计、管理员导出 CSV、班次 Web
- 群离返岗静默落库

当前主要痛点：

- 每次打卡都要输入 4 行配文，重复劳动高
- 注册与管理员权限依赖人工维护，清库后恢复麻烦
- HR 需要手动导出二次加工，异常信息不够聚焦
- 离返岗与排班表「请假/出差」标记尚未打通
- 失败提示粒度较粗（时间不一致/姓名不一致）

## 2. 项目目标

### 员工侧

- 点按钮后只发图即可打卡，减少重复输入
- 打卡失败原因更清晰

### HR / 组长侧

- 仅在异常时推送，减少噪音
- 支持排班表样式的 Excel 日历导出

### 管理侧

- 支持工号自助注册
- 管理员权限与 Google 表联动、自愈恢复

### 建议指标

- 打卡输入从 4 行降到 0 行（零配文）
- 工号自助注册可用
- HR 每日处理时长减少 50%
- 零配文打卡占比 >= 80%（上线 2 周后）

## 3. 用户与场景

| 用户 | 典型场景 |
|---|---|
| 员工 | 群里点“签到/签退”后直接发截图 |
| 新员工 | 私聊 Bot 发工号完成注册 |
| HR | 每天只看异常名单并导出月历 |
| 组长 | 仅在本组异常时接收提醒 |
| 管理员 | 清库后通过同步自动恢复权限 |

## 4. 里程碑需求

### M2.1 零配文打卡（P0）

**目标**：已注册员工点“签到/签退”后，只发截图即可完成打卡。

**方案**：

- 使用 `checkin_action_session` 记录用户待执行动作（签到/签退）
- 图片消息可无 caption
- 工号、英文名读取注册表；事项读取 pending action
- 旧版 4 行配文继续兼容

**验收**：

- 已注册 + 点按钮 + 合法截图 -> 成功
- 未点按钮且无配文 -> 忽略或提示先点按钮
- 旧流程不回归

---

### M2.2 注册与管理员联动（P0）

**目标**：降低注册门槛、减少管理员人工维护。

**方案**：

- 私聊支持“纯工号注册”
- 工号必须存在于排班表，否则拒绝
- 保留旧注册格式（英文名$工号）
- Google 表职位列同步管理员权限到 `admin_list`
- 支持固定白名单工号长期管理员

**验收**：

- 表外工号不可注册
- 同步后权限正确
- 清库后可通过同步恢复管理员

---

### M2.3 异常推送 + Excel 日历导出（P1）

**目标**：让 HR 从“主动翻查”变为“被动接收异常 + 一键导出可用报表”。

**异常推送**：

- 仅在有异常（缺卡/迟到/早退/未返岗）时推送
- 支持配置接收人（HR、组长、管理员）
- 无异常默认不推送

**Excel 日历导出**：

- 在 CSV 基础上新增 `.xlsx` 月历视图
- 行为员工，列为 1-30/31 日
- 状态色块与 Google 表视觉对齐

**验收**：

- 有异常才推送
- 导出结果可直接给 HR 使用

---

### M2.4 离返岗增强 + 表内请假/出差标记（P1）

**目标**：离返岗流程更顺滑，计划性请假状态与统计导出统一。

**方案**：

- 保留现有 `#离岗报备` / `#返岗报备`
- 增强一键开/闭合离岗记录
- 离岗 > 30 分钟说明必填
- Google 表中的“请假/出差”标记同步到统计与导出

**验收**：

- 表内标记与导出状态一致
- 未返岗可识别并计入异常
- 大于 30 分钟离岗规则生效

## 5. 本期不做

- 按 Google 每日格子（WG/WO）动态切换同一人当天班次
- 手势/人脸等非截图打卡
- QC 全流程恢复
- Docker 一体化部署
- 私聊离岗审批流恢复

## 6. 里程碑与上线计划（总工期：6 个月）

| 阶段 | 范围 | 说明 |
|---|---|---|
| 第 1-2 月 | M2.1 零配文打卡 | 需求冻结 + 开发联调 + 小范围内测 |
| 第 3-4 月 | M2.2 注册与管理员联动 | 工号注册、管理员同步、权限回归测试 |
| 第 5 月 | M2.3 异常推送 + Excel 日历导出 | HR 试运行，口径对齐与报表优化 |
| 第 6 月 | M2.4 离返岗增强 + 请假/出差标记 | 全链路回归、灰度上线、正式发布 |

建议顺序：`M2.1 -> M2.2 -> M2.3 -> M2.4`

- `register_complete`
- `export_trigger`
- `anomaly_push_sent`
- `sheets_sync_ok` / `sheets_sync_fail`

## 8. 风险与缓解

| 风险 | 缓解措施 |
|---|---|
| 零配文后误操作变多 | pending 超时、按钮引导、保留旧配文 |
| TG 压缩导致识图波动 | 失败重试、截图规范、必要兜底 |
| Google 表格式变更 | 同步前校验 + 告警 |
| 导出口径与 HR 习惯不一致 | 第 5 月样表对齐并迭代 |
