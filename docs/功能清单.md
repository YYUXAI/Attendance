## 一、当前可用功能（产品版）

### 1. 打开菜单

| 场景 | 操作 |
|------|------|
| 私聊 | 发送 **`/start`** → 输入框下方常驻按钮 |
| 考勤群 | 发送 **`/start`** 或点底部 **签到/签退/离岗/返岗** → 弹出 Inline 操作按钮 |
| 命令 | 输入 **`/`** 仅显示 **`/start`** |

---

### 2. 私聊（所有用户）

| 按钮 | 功能 |
|------|------|
| **注册** | 输入 `英文名$工号` → 确认后绑定 Telegram |
| **我的信息** | 姓名、工号、班次时间；本月出勤/缺卡/迟到/早退（与导出同规则，按打卡+月休判定） |

> 「我的信息」不再展示「本月已审批通过假期」（报备休息已下线）。

---

### 3. 私聊（管理员）

| 按钮 | 功能 |
|------|------|
| **导出** | 收到**当天全部考勤群**汇总 CSV（私聊文件） |
| **班次** | 打开班次配置 Web App（需 HTTPS 公网，如 ngrok） |

非管理员点击导出/班次 → 提示无权限。

---

### 4. 考勤群（已注册用户）

#### 4.1 底部按钮 + Inline 菜单

| 操作 | 说明 |
|------|------|
| **签到 / 签退** | 点带 ↗ 按钮 → 模板填入输入框 → 发 **TIME.IS 截图** + 配文 |
| **离岗 / 返岗** | 点 ↗ → 填入 `#离岗报备` / `#返岗报备` 模板 → 发送后**静默落库**（无 Bot 回复） |

未注册：Inline 按钮会引导先私聊注册。

#### 4.2 打卡模板格式

```text
#打卡
英文名：xxx
工号：xxxxx
事项：签到   （或 签退）
```

**必须**在配文中写 `事项：签到` 或 `事项：签退`，否则**静默忽略**（不处理、不回复）。

#### 4.3 打卡校验规则

| 步骤 | 规则 |
|------|------|
| 前置 | 已注册；任意群均可发图打卡（`shift_id` / 组织可选，未配置时 `clock_records.shift_id` 可为空） |
| AI 关闭 | 直接记服务器时间，不校验截图 |
| AI 开启 | OCR 识别姓名、日期、时间（支持佛历 / TIME.IS 乱码容错） |
| 姓名 | 截图姓名须与注册一致；或配文 `英文名+工号` 与注册一致时可信任配文 |
| 日期 | 截图日期须与**发消息当天**一致（按北京时间） |
| 时间 | 截图时间与发送时刻偏差 ≤ `CHECKIN_AI_MAX_CLOCK_SKEW_MINUTES`（默认 30 分钟）；**入库打卡时刻 = 发消息时间**（非截图钟面时间） |
| 成功 | **静默**（不回复）；失败才回复「姓名/日期/时间不一致」等 |

---

### 5. 群内离岗 / 返岗（无审批）

| 项目 | 说明 |
|------|------|
| 触发 | 群内发送含 `#离岗报备` / `#返岗报备` 的消息 |
| 记录 | 写入 `temporary_leave_records` |
| 导出 | CSV 列「离岗时间」：已返岗显示 `HH:MM:SS-HH:MM:SS(时长)`；未返岗显示 **未返岗** |
| 日志 | `[LEAVE_RECORD]`、`[LEAVE_EXPORT]` |

> 与已下线的「私聊离岗报备 + 审批」不同，当前仅群模板 + 直接落库。

### 6. 班次配置 Web（管理员）

| 功能 | 说明 |
|------|------|
| 查看/编辑 | 当月工号、英文名、班次时段、上班、下班、**月休** |
| 上传 | Excel/CSV 批量导入 |
| 下载模板 | 推荐「发到私聊」（iPhone 更稳） |

访问：`SHIFT_WEB_APP_PUBLIC_URL/shift-app/`（需 ngrok 或自有 HTTPS 域名）。

---

### 7. 考勤统计与导出

#### 7.1 管理员私聊「导出」

- 汇总**当天**所有考勤群
- CSV 表头：**群名、工号、英文名、班次、上班时间、下班时间、离岗时间、状态**

#### 7.2 分群自动推送（默认 23:00）

- 各考勤群：文字汇总 + 本群 CSV
- 配置：`GROUP_DAILY_SUMMARY_TIME`、`GROUP_DAILY_SUMMARY_TZ`

#### 7.3 全局日报（可选）

- 配置开启后：定时向指定管理员私聊发送全局打卡 CSV
- 配置：`DAILY_ATTENDANCE_REPORT_*`

#### 7.4 统计状态

**正常**、**迟到**、**早退**、**迟到+早退**、**缺卡**、**月休**

#### 7.5 跨天班次判定（如 18:00 → 次日 03:00）

系统以 `下班时间 <= 上班时间` 识别跨夜班：

| 日历日 | 应打卡 | 判定要点 |
|--------|--------|----------|
| 班次开始日 | **仅上班卡**（evening 签到） | 缺下班卡**不算**缺卡 |
| 之后每个工作日 | **早上下班卡** + **晚上上班卡** | 仅当**昨日 evening 有签到**才要求今早签退 |
| 迟到/早退 | 上班 vs 配置上班时间；下班 vs 配置下班时间 | |

非跨天班次：当日需签到 + 签退，与配置时间比迟到/早退。

实现：`domain/daily_attendance_status.py` → `group_attendance_summary_service`

## 二、已下线功能

以下功能**代码仍在仓库**，但 **Handler / Worker 已注释**，**勿启用**：

| 原功能 | 模块 |
|--------|------|
| 报备休息（私聊请假） | `handlers/rest.py` |
| 私聊离岗报备（带审批） | `handlers/temporary_leave.py` |
| 审批人（休假/离岗审批） | `handlers/approval.py` |

下线机制：

- `infra/bot.py`、`main.py` 中 router / worker 已注释
- `notification_worker` 跳过 template_id **1000–1999**（审批）、**2000–2999**（QC）
- `handlers/menu.py` 对旧 Inline 按钮静默忽略（日志 `disabled_feature`）

**仍保留且易混淆：**

- 群内 **离岗/返岗**（无审批，`attendance_actions.py`）
- 班次 **月休** 字段（判定月休日，不是私聊请假）

## 四、部署与运维

### 启动（两个终端）

```text
终端 1：python main.py          # Bot + Worker + HTTP 8787
终端 2：tools\ngrok\ngrok.exe http 8787   # 或使用 scripts\start_ngrok.ps1
```

### 关键环境变量

| 变量 | 说明 |
|------|------|
| `TELEGRAM_BOT_TOKEN` | Bot Token |
| `SHIFT_WEB_APP_PUBLIC_URL` | ngrok HTTPS 根地址（无末尾 `/`） |
| `CHECKIN_AI_ENABLED` | 是否启用截图 OCR 校验 |
| `CHECKIN_AI_MAX_CLOCK_SKEW_MINUTES` | 截图时间允许偏差（默认 30） |
| `GROUP_DAILY_SUMMARY_TIME` | 分群汇总时间（默认 23:00） |

### 日志关键字

```text
CHECKIN_RECOGNIZE    # 打卡识别
CHECKIN_HANDLER      # 打卡 handler
LEAVE_RECORD         # 离岗落库
LEAVE_EXPORT         # 离岗导出
disabled_feature     # 已下线功能旧按钮
```

### 注意事项

- **单实例**：勿多开 `main.py`（8787 冲突、Bot 重复收消息）
- **ngrok 免费版**：重启后 URL 可能变化，需更新 `.env` 与 BotFather 域名
- iOS 下载班次模板：优先「发到私聊」，避免 ngrok 拦截页
