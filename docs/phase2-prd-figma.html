# 项目背景

考勤 Bot（@zpxinbot）一期已上线并在 UX 组生产使用，核心能力包括：

  • 群内 TIME.IS 截图打卡（线上大模型识图，姓名/日期/时间严格识图校验）
  • Telegram 注册绑定（英文名 + 工号）
  • Google 排班表同步 → employee_shift_config（按人固定班次 + 月休日）
  • 迟到 / 早退 / 缺卡 / 月休判定，开班汇总推送
  • 管理员私聊导出 CSV、班次 Web App、群离返岗静默落库
  • 每晚 23:00 按群统计推送

仍依赖人工、体验待优化之处：

  • 每次打卡需复制 4 行配文（英文名、工号、事项），重复劳动多
  • 注册与管理员权限靠人工脚本维护，清库后易丢失
  • HR 需主动导出、手工对表；异常信息埋在全员汇总里
  • 离返岗与排班表「请假 / 出差」标记未打通，计划休假需人工核对
  • 识图失败提示粒度粗（姓名/时间类错误文案易混淆）

# 项目目标

角色目标
员工点按钮 → 只发截图即可打卡；截图规范清晰可预期
HR / 组长有异常才推送；导出即排班表样式日历，少做二次 Excel
管理员工号自助注册、权限与 Google 表联动可自愈
研发 / 运维在一期架构上增量交付，不推翻识图与判定底座

成功指标（建议）

  • 单次打卡人工输入：4 行配文 → 0 行（仅发图）
  • 新员注册：支持仅发工号完成绑定（表内校验）
  • HR 每日处理时长：减少 50%（异常推送 + 日历导出）
  • 零配文打卡占比：≥ 80%（上线 2 周后）

用户典型场景

# 需求详述

M2.1 零配文打卡 P0

问题每次打卡需填写 #打卡 + 英文名 + 工号 + 事项，员工重复输入、易错
方案
  1. 用户点 Inline「签到 / 签退」→ 系统记录 pending_action（已有 checkin_action_session，需接入 handler）
  2. 用户仅发送截图（可无 caption 或 caption 为空）
  3. 工号、英文名从 registrations 读取；事项从 pending 会话弹出
  4. 旧版 4 行配文格式继续兼容；配文与注册一致时可作识图兜底（可选增强）
交互
点「签退」→ 发图（无字）→ 静默成功
点「签退」→ 超时 15 分钟未发图 → pending 失效，需重点按钮
识图要求不变：截图须含 TIME.IS 大钟 + 日期 + 身份区（Slack / TG 用户名）；支持 上午3:04 等中文 12 小时制
验收
  • 已注册 + 点按钮 + 合法截图 → 成功（零配文）
  • 未点按钮、无配文、无 pending → 静默忽略或提示「请先点签到/签退」
  • 旧配文打卡全流程不退化
埋点checkin_zero_caption=true/false、pending_action_expired

M2.2 注册与管理员联动 P0

问题注册需 英文名$工号 格式；管理员靠脚本写入 admin_list，清库易丢
注册
  • 私聊发送纯工号（如 72357）→ 查 Google 同步的 employee_shift_config
  • 工号在表内 → 自动带出英文名，确认后绑定当前 Telegram
  • 工号不在表内 → 拒绝并提示联系 HR
  • 保留原 英文名$工号 注册方式
管理员
  • Google 表增加「职位 / 角色」列（如 组长、HR、管理员）
  • 同步任务写入 admin_list；支持配置白名单工号恒为管理员
  • 导出 / 班次 Web 权限与 admin_list 双校验不变
验收表外工号无法注册；同步后指定工号自动获得管理员；清库 + 重同步可恢复权限

M2.3 异常推送 + Excel 日历导出 P1

问题每晚汇总信息量大；HR 导出 CSV 后还要手工做成排班表样式月历
异常推送
  • 触发：每日班次结束后（或 23:00 统计后）仅当存在异常时推送
  • 异常类型：缺卡、迟到、早退、离岗未返岗
  • 接收人：可配置（HR 私聊 / 组长私聊 / 群管理员）
  • 无异常：不推送（或可选「今日全绿」摘要，默认关闭）
Excel 日历导出
  • 在现有 CSV 导出基础上增加 .xlsx 月历视图
  • 行：员工（工号、英文名、班次）；列：1–30/31 日
  • 单元格：状态色块（正常 / 迟到 / 早退 / 缺卡 / 月休 / 请假等，对齐 Google 表视觉）
  • 支持「今日 / 本周 / 本月」与管理员私聊导出入口
验收有 1 人缺卡即推送对应名单；Excel 与 Google 表列顺序、色块语义可对齐演示

M2.4 离返岗增强 + 表内请假/出差标记 P1

问题离返岗仅群模板静默落库；排班表上 ●请假 / 出差 与导出状态未联动
离返岗
  • 保留现有 #离岗报备 / #返岗报备 群模板
  • 增强：点「离岗」→ 可选一键开记录（时间取发送时刻）；点「返岗」→ 自动闭合最近未返岗记录
  • 离岗 > 30 分钟：说明必填（一期规则延续）
  • 导出「离岗时间」列：已返岗 HH:MM-HH:MM(时长)；未返岗显示「未返岗」
请假 / 出差
  • Google 表日格标记（●请假、出差等）同步到判定层
  • 导出与每晚统计中展示为「请假 / 出差」，不计缺卡
  • 与月休规则并列：计划休假免打卡
验收表内标记日与导出状态一致；离返岗一键流程写库正确；>30 分钟无说明被拦截
本期明确不做 / 暂缓
· 按 Google 每日格子（WG/WO）动态切换同一人当天班次（一期已按人固定班次，不做日格排班）
· 手势 / 人脸等非截图打卡
· 恢复 QC 质检全流程
· Docker 一体化部署
· 私聊离岗审批流（已下线，维持群模板方案）
